# دليل إطلاق منصة إهدنا على GitHub Pages — خطوة بخطوة

المدة المتوقعة: 15 دقيقة. النتيجة: رابط حي مثل `https://isma-ihdina.github.io` يعمل من أي جهاز وهاتف في العالم.

## الخطوة 1 — إنشاء حساب GitHub (إن لم يوجد)
1. افتح github.com واضغط **Sign up**
2. استخدم بريد المؤسسة، واختر اسم مستخدم مؤسسياً مثل: `isma-ihdina`
3. أكد البريد الإلكتروني

## الخطوة 2 — إنشاء المستودع (Repository)
1. اضغط زر **+** أعلى اليمين ثم **New repository**
2. اسم المستودع (مهم حرفياً): `isma-ihdina.github.io`
   (استبدل isma-ihdina باسم المستخدم الذي اخترته + `.github.io`)
3. اجعله **Public** ثم اضغط **Create repository**

## الخطوة 3 — رفع الملفات
1. داخل المستودع الجديد اضغط **uploading an existing file**
2. افتح مجلد `ihdina-site` على جهازك، وحدد **كل ما بداخله** (وليس المجلد نفسه) واسحبه إلى صفحة الرفع — بما فيها مجلد platform وملف .nojekyll
3. اكتب في خانة الوصف: `الإطلاق الأول` ثم اضغط **Commit changes**
4. انتظر اكتمال الرفع (الحجم 1.4 ميغابايت فقط — دقائق قليلة)

## الخطوة 4 — تفعيل GitHub Pages
1. داخل المستودع: **Settings ← Pages** (من القائمة الجانبية)
2. تحت Branch اختر **main** ثم **Save**
3. انتظر دقيقتين — سيظهر الرابط: `https://isma-ihdina.github.io`

## الخطوة 5 — التحقق
- افتح الرابط: تظهر الواجهة التسويقية بالشعار
- جرّب زر تبديل اللغة (English / Deutsch)
- افتح المنصة التفاعلية: `https://isma-ihdina.github.io/platform/`
- جرّب من هاتفك — يعمل في كل مكان

## الخطوة 6 (لاحقاً) — ربط النطاق الخاص
عند شراء النطاق (المتاح حالياً: **ihdina.info** أو **isma-ihdina.org** أو **ihdina-global.org**):
1. في Settings ← Pages ← Custom domain: أدخل النطاق واحفظ
2. عند مسجّل النطاق أضف سجل CNAME يشير إلى `isma-ihdina.github.io`
3. فعّل **Enforce HTTPS**

## التحديثات مستقبلاً
أي تعديل: ارفع الملف المعدل بنفس طريقة الخطوة 3 — يُحدَّث الموقع تلقائياً خلال دقيقة.
